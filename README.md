# منظومة التوظيف الذكية متعددة الأبعاد

> تحويل التوظيف من عملية إدارية إلى رحلة حياة مهنية ذكية، مستدامة، ومتطورة — حيث يكون الفرد محور المنظومة، والذكاء الاصطناعي هو المرشد الحيوي.

الواجهة الأمامية (Web) لمنصة توظيف متعددة الأبعاد تجمع الباحثين والجهات والموصين والمدراء في نظام بيئي واحد.

---

## 🛠️ التقنيات

| الطبقة | التقنية |
|--------|---------|
| إطار العمل | **Vue 3** (Composition API, `<script setup>`) |
| اللغة | **TypeScript** (الوضع الصارم) |
| أداة البناء | **Vite 5** |
| واجهة المستخدم | **Vuetify 3** |
| إدارة الحالة | **Pinia** |
| التوجيه | **Vue Router 4** |
| طلبات HTTP | **Axios** (طبقة العقد) + **Supabase** (محوّل مؤقّت/مرجع) |
| التعدد اللغوي | **vue-i18n** (عربي افتراضي + إنجليزي، RTL) |

نظام الألوان مستمدّ من وثيقة الـ UI/UX: أزرق داكن `#1A365D`، فيروزي `#319795`، برتقالي `#ED8936`.

> **📐 قرار المعمارية والتسليم**: المشروع يُحوَّل ويُسلَّم على مكدّس الفريق (Vue SPA + **NestJS + JWT** في [`api/`](api) + Docker/Nginx + Tailwind)، ومصدر الحقيقة للتكامل هو [`api/openapi.yaml`](api/openapi.yaml). الخطة الكاملة في **[`DOC/ARCHITECTURE.md`](DOC/ARCHITECTURE.md)**.

## 🚀 التشغيل

### Docker (موصى به — كل شيء بأمر واحد)

الدومين المعتمد: **https://recruitment.mawazinswift.com**

```bash
cp .env.docker.example .env
docker compose up -d --build
```

ثم اربط nginx على المضيف (ملف جاهز في `docker/host-nginx/recruitment.mawazinswift.com.conf`) وفعّل الشهادة:

```bash
# مثال — انسخ الكونفيج لمجلد nginx عندك ثم:
sudo certbot --nginx -d recruitment.mawazinswift.com
sudo nginx -t && sudo systemctl reload nginx
```

التطبيق داخليًا على `127.0.0.1:8080`؛ العامة عبر الدومين فقط.

| الخدمة | الدور |
|--------|------|
| `web` | واجهة Vue عبر nginx + بروكسي `/api` و WebSocket |
| `api` | Laravel (FrankenPHP) |
| `queue` | عامل الطابور |
| `reverb` | البثّ اللحظي |
| `mysql` / `redis` | قاعدة البيانات والكاش |

إيقاف: `docker compose down` — مع الإبقاء على البيانات: بدون `-v`.

### تطوير محلي بدون Docker

```bash
npm install
npm run dev        # خادم التطوير على http://localhost:5273
npm run build      # بناء الإنتاج
npm run typecheck  # فحص الأنواع
```

> **وضع التجربة:** لا يوجد Backend بعد — المصادقة والبيانات محاكاة (Mock) لكنها **تفاعلية بالكامل**: الإجراءات (تقديم، حفظ، قبول رغبة، تغيير حالة، إضافة مهارة، نشر فرصة...) تُحفظ في `localStorage` وتنعكس عبر الشاشات. في صفحة الدخول أدخل أي بريد/كلمة مرور.
> لتجربة الأدوار المختلفة اكتب داخل البريد: `company` (جهة توظيف)، `admin` (مدير)، `endorser` (موصٍ)، وإلا فالدور باحث.

### التفاعل الحقيقي (مخازن Pinia محفوظة محليًا)

| المخزن | ماذا يفعل |
|--------|-----------|
| `Applications` / `Saved` | التقديم يُضاف لطلباتي؛ حفظ الفرص (bookmark) |
| `Wishes` | قبول/رفض/تفاوض يُحدّث العدّادات |
| `Notifications` / `Messages` | تعليم كمقروء + شارات حيّة؛ إرسال رسائل يبقى |
| `Candidates` | تغيير حالة الترشيح + إجراءات جماعية |
| `Profile` / `Resumes` | إضافة/حذف مهارات وخبرات؛ حفظ سير من المعالج |
| `PostedOpportunities` / `Surveys` | نشر فرص ومسودات؛ منشئ استبيان كامل |
| `Endorser` | إضافة توصية تنقل الطلب للمقدّمة |
| `Profile` (مهارات موثّقة) | كل مهارة بإثباتات (اختبار/توصية/مشروع/شهادة) → ثقة ومستوى AI |
| `Trust` | نسبة ثقة (8 عوامل موزونة) + رادار + نصائح AI |
| `Interviews` | مقابلات AI تفاعلية → نتيجة تُحدّث نسبة الثقة |

> **طبقة AI مجرّدة** في `src/services/ai/` (محاكاة الآن، قابلة للتبديل لـ Claude API عبر مفتاح واحد) تُشغّل: مستوى المهارة، نصائح الثقة، أسئلة/تقييم المقابلة، تحليل الفيديو.

## 🗂️ الشاشات المنجزة (المرحلة 0)

| الدور | الشاشات |
|-------|---------|
| **عام (زائر)** | الصفحة التعريفية (Landing)، تسجيل الدخول، إنشاء حساب (اختيار دور) |

| **جهة التوظيف** | لوحة التحكم، نشر فرصة، الترشيحات + ملف المرشح، إدارة الرغبات، التحليلات |
| **الموصي** | لوحة الموصي، إضافة توصية (نص/صوت/فيديو/مستند + أسئلة AI) |
| **المدير** | لوحة المنصة، إدارة المستخدمين، الأدوار والصلاحيات (RBAC) |
| **مشترك** | الاستبيانات الثمانية |
