# خطة توحيد التصنيف (حيّة) — مصفوفة القطاعات المشتركة

> **الهدف:** اعتماد نموذج تصنيف موحّد صحيح ومنطقي وقابل للبناء عليه مستقبلًا لكل الأطراف
> (باحث/جهة/خبير)، مبنيّ على مصفوفة الإكسل `job_platform_classification_matrix.xlsx`
> مع الحفاظ على ما هو أقوى في اللوجيك الحالي (طبقة المهارات + `personas.ts`).
> المصدر المرجعي: ورقة العمل (21 قطاع · 96 تخصّص · 3 أطراف · 8 أنواع فرص · 7 قواعد حوكمة).

---

## المبدأ المؤسِّس: ثلاثة محاور متعامدة لا تختلط أبدًا

| المحور | يجيب عن | المصدر | الحالة |
|---|---|---|---|
| **1) مَن الطرف** (Identity) | من المستخدم؟ | `personas.ts` (شخصيّات باحث/أنواع جهات/تخصّصات خبراء/حوكمة) | ✅ موجود ومتوافق مع الإكسل — **يبقى كما هو** |
| **2) أي مجال** (Field) | في أي قطاع/تخصّص/مهارة؟ | **جديد:** `sectors.ts` — قطاع(21)→تخصّص(96)→مهارة | 🔨 يُبنى (يمتصّ `taxonomy.ts`) |
| **3) أي شكل فرصة** (Form) | ما شكل العلاقة؟ | حقل مستقل `OPPORTUNITY_TYPES` (8) | 🔨 يُوسَّع (من `RequestKind` الحالي) |
| **+ وسوم ثانوية** | تصنيف مزدوج حسب السياق | مصفوفة، قاعدة الحوكمة 6 | 🔨 يُضاف |

**القاعدة الذهبية:** القطاع **محور مشترك** — يختاره الباحث والجهة والخبير بنفس المفاتيح،
والمطابقة = `القطاع + التخصّص + المسمى + المهارات + الموقع + نوع الفرصة` (ورقتا 04/05).

---

## قرارات حاسمة (المعتمدة)

1. **اعتماد الـ21 قطاعًا كاملةً** — لا مجموعة جزئية (التجزئة تُعيد فجوة التغطية). إدارة الكثرة عبر
   **`priority` (أولوية الظهور)** لا البتر: أهمّ ~8 قطاعات ظاهرة + «المزيد».
2. **الإبقاء على العمق ثلاثي المستوى** (قطاع→تخصّص→مهارة) — لا تسطيح. طبقة المهارات الحالية
   (`categorizeSkill`/`ALL_SKILLS`) تُدمَج داخل التخصّصات لا تُحذف.
3. **«نوع الفرصة» حقل مستقل** — أبكر مكسب منطقي؛ العناصر التي وسمها الإكسل «نوع فرصة/وسم»
   (شراكة/شريك مؤسس/فرص تعاون تحت S20) تُوجَّه لهذا الحقل لا لشجرة القطاعات (قاعدة الحوكمة 3).
4. **`personas.ts` لا يُلمَس** — محور الطرف، مُصادَق عليه من ورقة 06.
5. **طبقة توافق (shim)** لـ`taxonomy.ts` — يُعاد تصديره من `sectors.ts` كي لا نكسر المستهلكين
   دفعةً واحدة (globalSearch/TaxonomyTree/ProfileStore/فلاتر المقيّمين…)، ثم نهجّرهم تدريجيًّا.
6. **ترميز قواعد الحوكمة السبع** كثوابت مرجعية + **فرض قاعدتين آليًّا:** منع الكلمات العامة
   كقطاعات (عمال/موظفين/تقني/متفرقات)، وإخفاء «أخرى/S21» افتراضيًّا.

## نموذج البيانات المستهدف (`src/services/sectors.ts`)

```ts
export interface SubSpecialty {
  id: string
  label: string; en: string
  type: 'specialty' | 'title' | 'sub_sector' | 'opportunity_tag' // نوع العنصر (ورقة 03)
  sampleTitles: string[]   // أمثلة مسميات/خدمات
  keywords: string[]       // كلمات بحث عربي+إنجليزي (تُغذّي globalSearch/autoClassify)
  skills: string[]         // ← مدموجة من taxonomy.ts الحالي
  fromMatrix: boolean      // «من التصنيف المرفق؟» — لقياس التغطية
  note?: string
}
export interface Sector {
  code: string             // 'S01' (مفتاح المصفوفة)
  id: string               // slug ثابت للتوافق الخلفي: 'technology'
  label: string; en: string
  icon: string; color: string
  priority: number         // أولوية الظهور 1..21
  description: string
  subs: SubSpecialty[]
}
export const SECTORS: Sector[]                 // 21 قطاعًا
export const OPPORTUNITY_TYPES: {...}[]         // 8 أنواع مستقلة
// APIs متوافقة (تبقى ثابتة للمستهلكين):
export function categorizeSkill(skill): string
export function getSector(code|id): Sector
export const ALL_SKILLS: string[]
export function migrateSector(oldId): string    // جدول old→new
```

### جدول المطابقة القديم→الجديد (أكبر مخاطرة — يُحسم في المرحلة 0)
القطاعات الحالية الثمانية → قطاعات الإكسل (بعضها ينقسم):
- `technology` → **S01** (1:1)
- `management` → **S02** إدارة+HR / **S03** مالية / **S04** مبيعات+تسويق (انقسام → افتراضي S02 + وسم للمراجعة)
- `design` → **S11** التصميم والفنون / جزء **S12** إعلام
- `engineering` → **S05** الهندسة والمقاولات
- `health` → **S08** الصحة
- `legal` → **S13** القانون والاستشارات
- `education` → **S09** التعليم والتدريب
- `logistics` → **S07** النقل والتوصيل
- **قطاعات جديدة كليًّا (كانت مفقودة):** S06 حرف وصيانة · S10 ضيافة ومطاعم · S14 أمن وسلامة ·
  S15 خدمات منزلية · S16 زراعة وبيئة · S17 رياضة ولياقة · S18 تصنيع وإنتاج · S19 عقار ومرافق ·
  S20 ريادة وشراكات · S21 أخرى.

## ما نكسبه بهذا التركيب
- **الآن:** تغطية سوق كاملة (حرفيّون/دليفري/ضيافة/عمالة… لا مجرّد ياقات مكتبية)، فصل نظيف بين
  «مَن/أي مجال/أي شكل»، بحث ثنائي اللغة أقوى.
- **مستقبلًا:** التوسّع بلا فوضى (قواعد الحوكمة)، مفاتيح مطابقة موحّدة تُسهّل أي محرّك توصية/AI لاحق،
  وطبقة بيانات واحدة يستهلكها كل الأطراف بدل تصنيفات متفرّقة.

## ما لا نفعله (صريحًا)
- لا نمحو `taxonomy.ts` فجأة (نُبقيه shim ونهجّر تدريجيًّا).
- لا نمسّ `personas.ts`.
- لا نعتمد مجموعة جزئية من القطاعات (نستخدم الأولوية لا البتر).
- لا نجعل أي كلمة عامة أو «نوع فرصة» قطاعًا.
