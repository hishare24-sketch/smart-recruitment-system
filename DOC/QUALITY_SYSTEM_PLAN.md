# خطّة نظام الجودة والاستشعار الذاتيّ (Self-Healing Quality System)

> ملفّ حيّ مربوط بالمشروع. المصدر الواحد للحقيقة لبناء نظام يكتشف الأخطاء/الأعطال/المشاكل،
> يصنّفها بقواعد ثابتة ومتغيّرة (توقيت/نوع/مكان)، يتتبّعها بذكاء، ويعرض إحصاءها في كنسول الأدمن —
> بأقلّ اعتماد على اختبار بشريّ. آخر تحديث تُضبط تلقائيًّا عند كلّ مرحلة.

---

## 0. المبادئ الحاكمة (ثابتة)
1. **بوّابة قبل كلّ دمج:** لا يُدمج كود يكسر typecheck/الاختبارات/البناء. آليّ 100%.
2. **طبقتان متكاملتان:** جودة وقت-البناء (CI) + رصد وقت-التشغيل (telemetry) — الأولى تمنع، الثانية تكتشف ما فات.
3. **الأتمتة تفعل بترتيب الثقة:** تمنع → تكتشف → تُبلّغ → **تُصلح الآمن فقط**. إصلاح المنطق يبقى بمراجعة ضيّقة.
4. **إعدادات منضبطة ومركزيّة:** كلّ العتبات/القواعد في مصدر واحد مُصرَّح (config)، لا أرقام سحريّة متناثرة.
5. **كلّ خطأ له بصمة:** يُجمَّع بالبصمة (نوع+مكان+توقيع)، فلا ضجيج ولا تكرار.

---

## 1. محرّك القواعد (Rule Engine) — قلب النظام

كلّ إشارة (خطأ/عطل/مشكلة) تمرّ بمحرّك يحسب **الخطورة + التوجيه + الإجراء** من قواعد **ثابتة** و**متغيّرة**.

### 1.أ القواعد الثابتة (Fixed)
| القاعدة | القيمة |
|--------|--------|
| CI فاشل | يمنع الدمج (خطورة قصوى) |
| خطأ render يُفرِّغ الصفحة (`blank`) | Critical فورًا |
| 5xx على مسار API | High |
| 401/419 (جلسة) | Info (متوقّع) — لا ينبّه ما لم يتجاوز عتبة |
| تغطية < العتبة | يمنع الدمج |
| نوع/type خطأ TS جديد | يمنع الدمج |

### 1.ب القواعد المتغيّرة (Dynamic) — تُعدَّل حسب:
- **التوقيت (when):** نافذة زمنيّة — «5 أخطاء من نفس البصمة خلال 10 دقائق» = تصعيد؛ فحص عميق ليليّ مجدول؛ تشديد أثناء ساعات الذروة.
- **النوع (what):** render / api-5xx / api-4xx / slow-request / console-warn / job-failed — لكلّ نوع خطورة وقناة أساس تُعدَّل ديناميكيًّا بالتكرار.
- **المكان (where):** طبقة (frontend-render / backend / api-client / queue) × نطاق (admin / public / seeker …) × موديول/مسار — يحدّد **مَن يعالج** (توجيه الوكيل) والخطورة (خطأ في الأدمن أو الدفع أخطر من صفحة عامّة).

الناتج لكلّ إشارة: `{ fingerprint, severity, layer, scope, module, route, rule_hits[], action }`.

### 1.ج مخرجات الإجراء
`ignore` · `track` · `alert(channel)` · `escalate` · `auto-fix-safe` · `dispatch-agent`.

---

## 2. التتبّع الذكيّ (Smart Tracking)
- **البصمة (fingerprint):** hash(type + normalized-message + top-frame + route). يجمع نفس الخطأ مهما تكرّر.
- **دورة حياة العنصر:** `new → ongoing → resolved (auto عند صمت N ساعة) → regressed (عاد بعد الحلّ)`.
- **عدّادات:** count · firstSeen · lastSeen · معدّل التكرار · عدد المتأثّرين · نافذة الاندفاع (spike).
- **تصنيف:** جديد / انحدار (regression) / معروف — يوجّه الأولويّة.

---

## 3. الوكيل الذكيّ متعدّد الطبقات (Multi-layer Agent)
يعيد استخدام بنية مزوّدات الذكاء الموجودة (`Modules/Ai`).
- **L1 — الفرز (Triage):** رخيص وقاعديّ — يصنّف بالبصمة/النوع/المكان، يطبّق القواعد، يزيل التكرار. بلا LLM.
- **L2 — التشخيص (Diagnose):** للأخطاء الحرجة/الجديدة فقط — LLM يحلّل الستاك + سياق الكود ويقترح السبب الجذريّ.
- **L3 — المعالجة (Remediate):** إصلاح آمن آليّ (تنسيق/تبعيّات) أو فتح PR/Issue بإصلاح مقترح **بموافقة بشريّة** (لا دمج تلقائيّ للمنطق).
- الحوكمة: كلّ استدعاء LLM محكوم بالحصص/المفتاح كبقيّة النظام؛ يعمل بلا مفتاح كتصنيف قاعديّ فقط.

---

## 4. لوحة الجودة في كنسول الأدمن
تبني على `HealthMetrics` الموجودة (`errorsToday/failedJobs/pendingJobs`) + الجداول الجديدة:
- **مجموعات الأخطاء:** بالنوع/المكان/الخطورة + الاتّجاه + آخر ظهور + حالة الحياة + تشخيص الوكيل المقترح.
- **حالة CI/الاختبارات:** آخر تشغيل، نسبة النجاح، اتّجاه التغطية (عبر GitHub API).
- **مركز التنبيهات:** التنبيهات المنظّمة وفق القواعد (ثابتة + متغيّرة).
